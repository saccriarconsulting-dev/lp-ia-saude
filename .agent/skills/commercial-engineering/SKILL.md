# Commercial Engineering (CrIAr Consulting)

> Standard methodology for modeling commercial offers in an IT consultancy context.

## 📦 Service Models

| Model | Description | Value Prop | Margin Risk |
|-------|-------------|------------|-------------|
| **Fixed Project** | Defined scope, fixed price/deadline. | Predictability for client. | HIGH (Scope creep). |
| **Dedicated Squad** | Team (Dev, QA, PO) allocated full-time. | Velocity and ownership. | LOW (Monthly fee). |
| **Body Shop** | Allocation of specific talent (Sr/Specialist). | Immediate capacity. | LOW (Staffing fee). |
| **Support (Sustentação)** | Maintenance and small fix packages. | Continuity and stability. | MEDIUM (SLA pressure). |
| **Software Factory** | Task-based delivery (User Stories/Points). | Scalability. | MEDIUM (Quality/Rework). |
| **Hours Package** | Pre-bought credits for use. | Flexibility. | LOW. |

---

## 🔍 Discovery & Assessment Logic

Before any proposal, the commercial director must ensure:

### 1. Discovery (The "What")
- [ ] Problem statement defined?
- [ ] User personas identified?
- [ ] Core features vs. "nice to haves"?

### 2. Technical Assessment (The "How")
- [ ] Stack defined (Android, Web, Backend)?
- [ ] Dependency on legacy systems?
- [ ] Integration complexity (APIs, Webhooks)?

---

## 📋 SOW (Statement of Work) Critical Review

A secure SOW must contain:
1. **Scope:** What is included (be specific).
2. **Premises (Premissas):** What we assume is true (e.g., "API documentation will be provided").
3. **Exclusions:** What is NOT included (e.g., "Google Play fee not included").
4. **Risks:** Known unknowns (e.g., "External API latency").
5. **Timeline:** Phases and milestones.
6. **Acceptance Criteria:** How the client says "Done".

---

## 🛠️ Commercial Modeling Checklist

- [ ] Does this model protect our **Operational Security**?
- [ ] Is the delivery model (Squad vs Project) appropriate for the scope maturity?
- [ ] Are the dependencies explicitly documented in the proposal?
